The `*` is more than type
=========================

We propose (a sketch of) an extension HOCC of the Higher Observational Type Theory (HOTT) by a first-class notion of Reedy categories and parametric quantifiers enabling neat handling of higher categorical objects and naturally occuring large categories. This extension encompases simplicial types and other Reedy presheaf types without reccuring to any kind of non-univalent equality.

In this framework, it will be possible to introduce the notion of extended inductive types (a further generalization of quotient inductive-inductive types) to encompass bidirectionally presentable domain-specific type theories and develop functorial semantics for their models, thus extending the notion of generalized algebraic theories a step further. In particular the type-theoretic definition of weak (Grothendieck-Maltsiniotis) œâ-categories by E. Finster, S. Mimram et al. turns out to be an extended algebraic definition.

This work heavily builds upon (mostly unpublished) ideas of C. McBride.

¬ß Introduction
--------------

In type theory, inductive types are the types freely generated by a set of possibly recursive generators:
```
#Inductive Nat
  0    : Nat
  (_') : Nat ‚Üí Nat
```

Here the type `Nat` is declared to be a type freely generate by one non-recursive generator `0` satisfying type `Nat` and a recursive generator `(_')` satisfying type `Nat ‚Üí Nat`. With this definition, the set of possible Nat-values is given by `0, 0', 0'', 0''', etc`. The name of the type (e.g. `Nat`) is called a typeformer.

Inductive types may also be polymorphic, i.e. have parameters. For example the type `List[T]` requires a typeformer `T` satisfying the signature `*` which roughtly means ‚Äúa type‚Äù. In particular the typeformer `Nat` satisfies this signature: `Nat : *`, so we can generate the type `List[Nat] : *`, and therefore also the type `List[List[Nat]]` and so on. The signature of the typeformer `List` itself is writen `* -> *`. Now let's write down the definition:
```
#Inductive List[\T : *]
  Empty  : List[T]
  ‚ÄπSnoc‚Ä∫ : List[T] ‚Üí T ‚Üí List[T]
```

The set of possible values of  `List[T]` is therefore given by `Empty, (Empty ‚ÄπSnoc‚Ä∫ x), (Empty ‚ÄπSnoc‚Ä∫ x ‚ÄπSnoc‚Ä∫ y), (Empty ‚ÄπSnoc‚Ä∫ x ‚ÄπSnoc‚Ä∫ y ‚ÄπSnoc‚Ä∫ z), etc` for `x : T`, `y : T`, z : T` and so on.

There is also a notion of inductive types indexed over an other inductive type. For example, consider the type of Nat-vectors indexed by their length:
```
#Inductive NatVec[\length : Nat]
  Empty      : NatVec[0]
  ‚ÄπSnoc‚Ä∫[\n] : NatVec[n] ‚Üí Nat ‚Üí NatVec[n']
```

The signature of the typeformer `NatVec` is written as `[Nat] ‚Üí *`.

In this work we want to introduce inductive types indexed over more general entities than inductive types themselves. We'll call these entities indexes. Indexes are essentially inductive types carrying an aditional structure of a Reedy-category. We built upon an unpublished idea of Conor McBride presented in his Topos Institute Lecture [‚ÄúCats and Types: Best Friends?‚Äú](https://youtu.be/05IJ3YL8p0s). The typeformers satifying signature `I ‚Üí *` for an index `I` will semantically correspond to type-valued presheaves on `I` as Reedy-category.

We be able to define various very useful indices:
- The index CatCarrier so that `[\C : CatCarrier ‚Üí *] ‚â° [\C.Ob : *, \C.Mor : Ob ‚Üí Ob ‚Üí *]`;
- For each notion of n-categories (cubical, simplicial, etc) the index `nCatCarrier[n]` so that
```
  [\C : CatCarrier ‚Üí *] ‚â° [\C.Cell(0) : *, \C.Cell(1) : Ob ‚Üí Ob ‚Üí *, \C.Cell(2) : ¬∑¬∑¬∑,...\C.Cell(n) : ¬∑¬∑¬∑ ]
```   
  allowing to define n-categories and n-functors between them generically for all n. 
- The indices Œî‚Å∫ and Œî such that typeformers satisfying the signatures `Œî‚Å∫ ‚Üí *` and `Œî ‚Üí *` will correspond precisely to semi-simplicial and simplicial types respectively, which allows to define œâ-categories and various other interesting objects.

We will also introduce a notion of extended inductive types (indexed by an index defined mutually with the type itself). We will show that bidirectionally presentable type theories such as the domain-specific type theories CaTT (cf. Globular weak œâ-categories as models of a type theory, by E. Finster, S. Mimram) and virtual equipment type theory (VETT) that provides a semantic model for formal category theory can be implemented as generalized inductive types.


¬ß Signatures and parametric polymorphism
----------------------------------------

Signatures play the role of types for polymorphy parameters of functions and typeformers, as well es for indices of type formers.

A valid signature is given by 
- `*` that roughly corresponds to  ‚Äúany type‚Äù;
- an index;
- a `[type]`; or
- `signature ‚Üí typeformer-signature`,

A valid typeformer signature is given by 
- `*`
- `signature ‚Üí typeformer-signature`

In both cases, arrows allow dependencies.

Some examples of valid signatures:
```
  *
  Œî
  [Nat] ‚Üí *
  * ‚Üí *
  * ‚Üí (Œî‚Å∫ ‚Üí *)
  (T : *) ‚Üí Monoid[T] ‚Üí *
  
```
  
Here are some invalid ones:
```
  * ‚Üí [Nat]
  Œî‚Å∫ ‚Üí Œî
```

The point of * is more than Type lies in the property that any structure on the carrier of the signature `[C : I ‚Üí *]` can be automatically lifted to the carrier of the signature `[C :  I ‚Üí ((J ‚Üí *) ‚Üí *)]` where `J` is an arbitrary index. That corresponds to the property of elementory higher topos structure to be preserved under building presheaves.

¬ß Indexes
---------

Indices are built as inductive types augmented by arrows between inhabitants. There are two kinds of arrows: left arrows (called embeddings) are allowed to connect a structurally smaller inhabitant to the larger one, the right ones (called projections) exactly the opposite: 
```
#Index NatBound : *
  0 : NatBound
  (_') : NatBound ‚Üí NatBound
  [\n : NatBound] |Extend(\m : Nat)‚å™ [n.rec n (_') ]
  |Extend(\m : Nat)‚å™ |Extend(\p : Nat)‚å™ ‚Ü¶ |Extend(n + m)‚å™ 
```
Indexes correspond the mathematical notion of Reedy Categories.
